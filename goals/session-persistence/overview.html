feat/session-persistence

Session Persistence

Denied sessions stay alive. The agent revises. The same session updates in place. No more starting over.

User Experience

Before

  1. 1 You review a plan, leave annotations, deny
  2. 2 Session dies — completion screen, done
  3. 3 Agent revises and resubmits
  4. 4 Brand new session — new tab, no context, no diff
  5. 5 Start over from scratch

After

  1. 1 You review a plan, leave annotations, deny
  2. 2 Banner: "Waiting for agent to revise..."
  3. 3 Agent revises and resubmits
  4. 4 Same session updates — diff shows what changed
  5. 5 Review again — approve or deny with more feedback

Works across all modes

📋

Plan Review

Agent revises the plan, session shows plan diff

🔍

Code Review

Agent makes changes, session refreshes with new diff

✏️

Annotate

Agent edits the file, session refreshes with updated content

What stays the same — Approve, exit, auto-close, and standalone sessions all work exactly as before. Only the deny→resubmit path changes. Sessions expire after 10 minutes if the agent doesn't come back.

What does NOT persist

✗ URL-based annotations
✗ "Annotate last message" sessions
✗ Archive sessions (read-only)
✗ Standalone / demo sessions

Recap

  1. Denied sessions stay alive instead of dying
  2. Agent resubmits → same session updates in place
  3. Works for plan, code review, and file-based annotate
  4. Matching by project+slug, project+branch, or filepath
  5. 10-minute timeout if the agent doesn't come back
  6. Agent doesn't need to know — matching is server-side
  7. Shared createDecisionCycle eliminates duplication
  8. Frontend shows amber "waiting" banner with cancel
  9. No changes to approve, exit, or standalone flows
