# Homepage redesign — cards × current

Round 2. You liked Option C (project cards); the rest were worse than today. These
5 are hybrids of **C (cards)** and the **current** flow. Open `index.html` to flip
between them (and a faithful **Current** mock to A/B against).

## The 5 hybrids
| | Name | Blend | What it keeps from today |
|---|---|---|---|
| C1 | Selectable cards | card grid + multi-select | the select → Launch-bar (Review/Annotate) batch flow |
| C2 | Cards + drill-down | card grid + inline expand | worktrees + PRs per project, each launchable |
| C3 | Resume + cards | active list on top, cards below | the active-sessions list, elevated to a "resume" zone |
| C4 | Mini-dashboards | rich cards | each project's own sessions, inline |
| C5 | List-cards | today's list density, card polish | the vertical list + per-row actions (smallest leap) |

## Read
- **C5** = lowest-risk evolution of what you have (same shape, nicer + actionable).
- **C1** = keeps your multi-select batch launch but in cards.
- **C3** = best if "resume" matters most (separates resume from start).
- **C4** = richest, best with few projects.
- **C2** = if worktree/PR drill-down is central.

Static mocks only. Pick one (or a hybrid) and I'll build it.
